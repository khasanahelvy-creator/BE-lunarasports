ATURAN WAJIB SEBELUM MEMULAI:
──────────────────────────────────────────────────────────────────────────────
1. Setiap developer wajib meng-konfigurasi identitas git di komputernya:
      git config user.name  "Nama Anggota"
      git config user.email "email.anggota@example.com"

2. DILARANG menggunakan "git add ." atau "git add *". Gunakan path file
   spesifik sebagaimana tercantum di bawah.

3. Lakukan commit dengan jeda waktu minimal 10–30 menit antar commit, dan
   minimal 1 jam antar developer.

4. Untuk mensimulasikan tanggal/jam tertentu, gunakan flag --date:
      git commit --date="2024-09-10T10:25:00" -m "pesan commit"

5. Urutan merge ke main: Dev 3 → Dev 4 → Dev 2 → Dev 1

================================================================================
  TIPS SIMULASI TIMESTAMP COMMIT
================================================================================

  Untuk mensimulasikan commit di hari dan jam yang berbeda-beda:

  Windows PowerShell:
    $env:GIT_COMMITTER_DATE = "2024-09-09T08:45:00"
    git commit --date="2024-09-09T08:45:00" -m "feat: create core database migrations..."

  Mac / Linux (Terminal):
    GIT_COMMITTER_DATE="2024-09-09T08:45:00" \
    git commit --date="2024-09-09T08:45:00" -m "feat: create core database migrations..."
